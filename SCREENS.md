## Adaptive Layout Guidelines

### Breakpoints
```
Mobile:   < 600px  (phone portrait)
Tablet:   600-900px (tablet, phone landscape)
Desktop:  > 900px  (laptop, desktop)
```

### Layout Patterns

**Navigation:**
- Mobile: Bottom navigation bar (< 600px)
- Tablet: Bottom navigation or top tabs (600-900px)
- Desktop: Sidebar (> 900px)

**Content:**
- Mobile: Single column, full width
- Tablet: 2 columns for lists, single for forms
- Desktop: Multi-column, sidebars, split views

**Modals:**
- Mobile: Full-screen or bottom sheet
- Tablet: Bottom sheet or centered modal (max 80% height)
- Desktop: Centered modal (max 600px width)

**Images:**
- Mobile: Full width in forms, 80x80px in lists
- Desktop: 400x600px in forms, 200x280px in grids

### Responsive Behavior

**Browse Grid:**
```
Mobile:   1 column (vertical list)
Tablet:   2-3 columns
Desktop:  4-6 columns
```

**Search Filters:**
```
Mobile:   Full-screen modal
Tablet:   Bottom sheet (50% height)
Desktop:  Inline sidebar
```

**Detail View:**
```
Mobile:   Full-screen page
Tablet:   Modal (80% height)
Desktop:  Sidebar (right) or modal
```

## Accessibility Considerations

### Screen Reader Support
- All images have descriptive alt text
- Form fields have proper labels
- Buttons have semantic meanings
- Navigation announcements

### Keyboard Navigation (Desktop)
- Tab order follows visual flow
- Focus indicators visible
- Escape closes modals
- Enter submits forms
- Arrow keys navigate lists

### Touch Targets (Mobile)
- Minimum 44x44px tap targets
- Adequate spacing between interactive elements
- Swipe gestures have fallback buttons

### Color Contrast
- WCAG AA compliant (4.5:1 for text)
- AAA for critical actions (7:1)
- Don't rely on color alone for meaning

### Font Scaling
- Support system font size preferences
- Test at 200% zoom
- Avoid fixed pixel sizes
